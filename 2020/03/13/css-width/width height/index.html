css-width/width height

width与height

width: auto

​ (一) 作用

​ 1.充分利用可用空间

​ 2.收缩与包裹（浮动、绝对定位）–>fit-content

​ 3.收缩到最小。table-layout中“一柱擎天”，中文可以断，英文单词不能断 –>min-content

​ 4.超出容器限制。

（二）无宽度，无图片，无浮动

​ (三)已设定好的width的弊端：流动性丢失，与现实世界表现不一样（页面布局错位）

css流体布局下的宽度分离原则

width属性不能与影响宽度的padding,border属性共存 => 让width独占一层标签，剩下的利用流动性在内部自适应呈现

用法：给原标签嵌套一层父标签，将width属性写在父标签上，padding等属性写在自己身上

好处：计算简便，页面结构更加稳固。

不好处：需要额外嵌套标签

解决方法：用box-sizing

box-sizing

支持content-box和border-box，不支持margin-box

height: 100%

对于width属性，当width为auto时，百分比值适用；而对于height属性，当父元素的height为auto，只要子元素在文档流中，其百分值就被忽略

因此，百分比高度想要起作用，其父集必须有一个可以生效的高度值

==如何支持==

1.设定显示的高度值

2.使用绝对定位

box-sizing属性？

用来控制元素的盒子模型的解析模式，默认为content-box
context-box：W3C的标准盒子模型，设置元素的 height/width 属性指的是content部分的高/宽
