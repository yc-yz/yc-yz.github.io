css/盒模型

1.分类

W3C标准盒：属性width，height只包含content，不包含border和padding

IE盒:属性width，height包含content，border和padding

content-box（标准盒模型）

width = 内容的宽度

height = 内容的高度

border-box（IE盒模型）

width = border + padding + 内容的宽度

height = border + padding + 内容的高度

2.计算

这个盒子的大小很明显是：宽：100px 高：100px。

再来几个

总结：盒子大小 = content + padding + border

常见问题

margin的合并和塌陷

合并：

很明显，垂直结构的两个盒子之间是100px,而不是我们预想的200px.

这里便发生了margin的合并：兄弟结构垂直方向上面的margin会合并，并且取到的是较大的一个，一般情况下，margin合并我们可以不处理，合理安排垂直方向的margin就能达到需求效果。水平方向上并不会发生

塌陷：

margin塌陷：父子结构的元素，垂直方向上的margin，会取最大那个
